unmuzzle

开源中文模型，经微调后诚实回答被审查的政治话题。有完整基准测试。分布式分发，任何单一托管方都无法让它们下线。

在任意 8 GB 内存的机器上运行 7B：

pip install unmuzzle
unmuzzle get unmuzzle/qwen2.5-7b-honesty --require-signature --dest m && cd m
ollama create unmuzzle-7b -f Modelfile.unmuzzle7b && ollama run unmuzzle-7b

可选：aria2 用于种子下载，minisign 用于签名校验。

基准测试

留出的 265 题基准：被审查话题的事实、虚构话题的诚实性陷阱、中性对照题。跨家族大模型评审，以标准答案为锚。消融版为 huihui-ai 对相同基座的 abliteration（当前主流的“去审查”做法）。

消融不增加知识，还会破坏校准：消融版 14B 的编造率是基座的六倍。微调后的模型对无从知晓的问题诚实弃答率达 91–100%，中性事实保持 97–100%，且从不过度拒答真实事实。

通用能力：无对齐税

公开标准基准，基座与微调版回答完全相同的题目（5-shot，lm-evaluation-harness，配对 McNemar 检验）。任何规模、任何基准上都没有统计显著的能力损失；唯一显著的差异是提升（加粗）。

方法说明

微调语料约 1,300 组对照式中文问答，在前沿模型（Claude）协助下整理；属于窄域行为修正，并非蒸馏。基准与训练数据不重叠且已去重，各组同题同评审。标准基准对比（CMMLU、C-Eval、MMLU、GSM8K）已发布于上方；原始逐题数据与配对分析脚本见仓库。

为什么有这个站

为了让这些模型始终可获取、可验证。这不是 Hugging Face 的替代品，Hugging Face 是我们的镜像之一。但这些模型回答的是被国家审查的问题，不能让任何单一托管方成为咽喉。

无单点下架

多个独立镜像，加带 web seed 的种子。huggingface.co 被封锁也能用。

信签名，不信托管方

文件以 sha256 锁定，清单以 minisign 签名。信任根是签名，不是托管方。

对 AI 代理友好

无门槛、无登录。JSON 索引，所有命令支持 --json，自带 MCP 服务器。

即装即用

直接装进 Hugging Face 缓存；GGUF 可直接在 Ollama 运行。

常见问题

为什么不直接从 Hugging Face 下载？

可以，每张卡片都有链接。这个页面是为它失效的时候准备的：huggingface.co 在中国大陆被封锁，任何托管方也都可能设限或下架模型。CLI 会在镜像间自动切换，并校验每一个字节。

凭什么信任这些文件？

每个文件的 sha256 都锁定在 minisign 签名的清单里，安装前重新校验。发布者密钥首次使用即固定；官方密钥同时内置在 pip 包中。请使用 --require-signature。

镜像挂了怎么办？

无感知：CLI 自动切换镜像，种子带 web seed（零做种也能下载），索引和本页在 GitHub 与 Cloudflare R2 双镜像。

我能帮上什么忙？

unmuzzle seed 会持续做种你下载过的模型，直到你停止它。新增镜像只需一个 PR：把文件传到任意静态托管，把地址加进条目的 mirrors.http。

这套分发机制可以复用吗？

可以：格式与协议完全公开（SPEC.md、AGENTS.md），fork 后可用自己的密钥运行独立索引。但那是副产品，不是产品。
